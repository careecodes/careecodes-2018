What's the difference between Git and GitHub?

Git is a version control system.
GitHub is a hosting service for Git repositories.
Version control systems are a category of software tools that help a software team manage changes to source code over time.1

Version Control is helpful for...

  • Collaboration
  • Source of truth and record
  • Maintaining multiple versions of the same software
Almost all software companies use a form of version control. And the ones who don‚Äôt, should!
There are a several different version control systems, but Git is the most popular one.2
There are a lot of ways to interact with a version control system, but a universal method that you'll encounter at almost any job is using the command line.

Vocab Time! ü§ì

Repository
Very simply, a repository is a bucket where your source code lives.
Git repositories contain all of your source code, plus the history of your source code.
Branch
  • A branch is essentially an independent line of development.
  • You can take advantage of branching when working on new features or bug fixes because it isolates your work from that of other team members.5

Let's learn some commands!

git init
Creates a new local repository
git clone
Creates a copy of a remote repository on your local machine.
git status
Tells you which files have changed, and files you still need to add to staging or commit.
git checkout
Changes you from one branch to another
git pull
Pulls changes from the remote repository into your local repository
git add
git add stages changes to be committed
staging is basically a holding area where you put files that you want to commit.
git add myFileName.js
git add .
git commit
  • Creates a commit and saves it to your repository
  • You have to add your changes to staging before you can commit
  • You must include a commit message when commiting changes.
git commit -m "This is my commit message!"
Commit messages are used to communicate...

To ask questions

To celebrate project milestones

To give a colleague constructive feedback

A good commit message is descriptive and well-formatted.3 Sometimes teams and companies have guidelines on how to format commit messages.
git push
This command ‚ÄúPushes‚Äù your local repository changes to your remote repository. You have to commit your changes before you can push!
git branch
  • Creates a new branch
git branch new-branch-name
git checkout -b new-branch-name
